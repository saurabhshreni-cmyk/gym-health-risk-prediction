2. Preprocessing

Merged multiple files by aligning on time (hourly level).

Dropped rows with missing values.

Created a risk label (0 = safe, 1 = risky) based on simple thresholds:

heart_rate > 160 bpm, or

steps > 8000 per hour, or

calories > 900 per hour.

Class distribution: about ~2500 safe rows, only ~3 risky rows.

This highlighted the class imbalance problem.

3. Handling Class Imbalance

Since only 3 risky examples existed, we applied upsampling of the minority class.

This artificially increased risky cases (e.g., upsampled to 200 samples) so that ML models could learn the difference.

Trade-off: Good for demo purposes, but may overfit due to lack of real risky data.

Trained multiple models on features (heart_rate, steps, calories) vs. risk.
